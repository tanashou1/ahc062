# AHC062 開発ルール

## 必須: ロジック変更後は必ず評価を実施すること

`src/main.rs` のアルゴリズム・パラメータを変更したら、**必ず** 以下を実行する:

```bash
./eval.sh
```

- スコアは `results/history.log` に自動追記される
- 最新の10ケース出力は `results/0000.txt` 〜 `results/0009.txt` に保存される
- スコアが下がった変更は revert すること

## 問題概要

- **N=200** のグリッド。各セル (i,j) に値 A[i][j] (1..N² の順列)
- **King's tour**: チェビシェフ距離=1 の隣接セルを辿り全セルをちょうど1回訪問
- **スコア**: `round( Σ_{t=0}^{N²-1} t × A[path[t]] / N² )`
  - 後に訪問するセルほど好感度貢献が大きい → **高A値を後に回す**のが最適

## 現在のアプローチ (src/main.rs)

1. **初期解**: Snake (蛇行) パス — 行ごとに左右交互に全セルを訪問
2. **局所探索**: シミュレーテッドアニーリング (SA) + 2-opt (セグメント逆転)
   - `path[l-1]` の King 隣接セルから `r` を探索 → 左辺の隣接を保証
   - 右辺 `king_adj(path[l], path[r+1])` を確認
   - delta = Σ_{t=l}^{r} (2t − l − r) × A[path[t]] (正 → 高A値が末尾へ)
   - 温度: 指数冷却 (5e7 → 1e3)
